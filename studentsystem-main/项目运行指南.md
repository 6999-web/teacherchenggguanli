# 学生综合信息服务平台 - 项目运行指南

> **技术栈**：前端 Vue 3 + 后端 FastAPI + MySQL 数据库

## 📋 前提条件

在运行项目前，请确保已安装以下软件：

- **Python 3.8+** (后端)
- **Node.js 18+** 和 **pnpm** (前端)
- **MySQL 8.0+** (数据库)

## 🚀 快速启动（三步运行）

### 第一步：启动数据库

#### 选项A：使用本地 MySQL
```powershell
# 1. 启动 MySQL 服务
# 在 Windows 上，可以通过服务管理器启动，或使用命令：
net start MySQL80

# 2. 创建数据库
mysql -u root -p
```

在 MySQL 命令行中执行：
```sql
CREATE DATABASE student_system CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;
```

#### 选项B：检查 MySQL 状态
```powershell
# 检查 MySQL 是否运行
Get-Service MySQL80

# 如果未运行，启动它
Start-Service MySQL80
```

### 第二步：启动后端服务器

```powershell
# 1. 进入后端目录
cd d:\student_system\backend

# 2. 激活 Python 虚拟环境（如果存在）
..\.venv\Scripts\Activate.ps1

# 或者创建新的虚拟环境
python -m venv .venv
.venv\Scripts\Activate.ps1

# 3. 安装依赖
pip install -r requirements.txt

# 4. 确认 .env 配置正确
# 检查 DATABASE_URL 是否匹配您的 MySQL 配置

# 5. 初始化数据库（首次运行）
python init_db.py

# 6. 启动后端服务器
python main.py
```

**后端启动成功标志**：
- 控制台显示：`Uvicorn running on http://0.0.0.0:8000`
- 数据库初始化成功
- 可访问：http://localhost:8000/docs (API 文档)

### 第三步：启动前端开发服务器

打开**新的终端窗口**：

```powershell
# 1. 进入前端目录
cd d:\student_system\frontend

# 2. 安装依赖（首次运行）
pnpm install

# 3. 确认 .env 配置
# VITE_API_URL=http://localhost:8000
# VITE_API_BASE_URL=http://localhost:8000/api

# 4. 启动开发服务器
pnpm run dev
```

**前端启动成功标志**：
- 控制台显示：`Local: http://localhost:5173/`
- 浏览器自动打开或手动访问上述地址

## 🔧 详细配置说明

### 后端配置文件 (`backend/.env`)

```env
# 数据库配置 - 请根据您的实际情况修改
DATABASE_URL=mysql+pymysql://root:password@localhost:3306/student_system

# JWT 配置
SECRET_KEY=your-secret-key-change-this-in-production
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=1440

# 文件上传
UPLOAD_DIR=./uploads
MAX_FILE_SIZE=10485760

# AI 配置（阿里云 Qwen）
DASHSCOPE_API_KEY=<DASHSCOPE_API_KEY>
QWEN_API_KEY=<QWEN_API_KEY>
QWEN_MODEL_NAME=qwen-plus
QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
QWEN_VL_MODEL=qwen-vl-max

# CORS
ALLOWED_ORIGINS=http://localhost:3000,http://localhost:8080,http://localhost:5173
```

**重要配置项说明**：
- `DATABASE_URL`: 修改 `root:password` 为您的 MySQL 用户名和密码
- `SECRET_KEY`: 生产环境请使用强随机密钥
- `ALLOWED_ORIGINS`: 根据前端端口添加对应的源

### 前端配置文件 (`frontend/.env`)

```env
VITE_API_URL=http://localhost:8000
VITE_API_BASE_URL=http://localhost:8000/api
```

**注意**：后端默认运行在 8000 端口，请确保前端配置匹配。

## 🐛 常见问题排查

### 问题1：数据库连接失败

**错误信息**：`ConnectionRefusedError` 或 `Can't connect to MySQL server`

**解决方案**：
```powershell
# 1. 检查 MySQL 服务是否运行
Get-Service MySQL80

# 2. 检查数据库是否存在
mysql -u root -p -e "SHOW DATABASES LIKE 'student_system';"

# 3. 检查 .env 中的数据库密码是否正确
# DATABASE_URL=mysql+pymysql://root:YOUR_PASSWORD@localhost:3306/student_system
```

### 问题2：后端端口冲突

**错误信息**：`Address already in use` 或 `OSError: [Errno 48]`

**解决方案**：
```powershell
# 查找占用 8000 端口的进程
netstat -ano | findstr :8000

# 终止进程（替换 PID 为实际进程 ID）
taskkill /PID <PID> /F

# 或者修改后端端口
# 在 main.py 中修改 port=8000 为其他端口
# 同时更新前端 .env 中的 VITE_API_URL
```

### 问题3：前端无法连接后端

**现象**：前端页面报 `Network Error` 或 `CORS` 错误

**解决方案**：
1. 确认后端已启动并运行在正确端口
2. 检查前端 `.env` 中的 API 地址是否正确
3. 检查后端 `.env` 的 `ALLOWED_ORIGINS` 包含前端地址
4. 清除浏览器缓存，刷新页面

### 问题4：前端依赖安装失败

**错误信息**：`pnpm install` 报错

**解决方案**：
```powershell
# 清除缓存
pnpm store prune

# 删除 node_modules 和 lock 文件
Remove-Item -Recurse -Force node_modules
Remove-Item pnpm-lock.yaml

# 重新安装
pnpm install
```

### 问题5：Python 包安装失败

**解决方案**：
```powershell
# 升级 pip
python -m pip install --upgrade pip

# 使用国内镜像源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 如果某个包冲突，尝试单独安装
pip install fastapi uvicorn sqlalchemy
```
